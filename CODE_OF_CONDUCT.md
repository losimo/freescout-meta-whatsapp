# Code of Conduct

[Català](CODE_OF_CONDUCT.ca.md) · [English](CODE_OF_CONDUCT.md) · [Castellano](CODE_OF_CONDUCT.es.md)

This project wants a community that is open and demanding at the same time. The general rule fits in one sentence: treat people with respect and ideas with rigor.

## Expected

- Good faith by default: assume the other person wants to improve the project.
- Direct, well-argued technical criticism, always about the code or the proposal, never about the person.
- Accepting disagreement: maintenance decisions are made with explained criteria, and a well-argued "no" closes the topic unless new arguments appear.
- Respecting other people's time: clear context, prior reading and concrete messages.

## Not accepted

- Personal attacks, put-downs, harassment or discrimination of any kind.
- Aggressive insistence after a decision has already been explained.
- Publishing other people's private information.
- Using the project as a battleground unrelated to maintaining the code.

## Scope and enforcement

This code applies to issues, PRs, discussions and any other communication linked to the project.

Violations can be reported privately to the maintainer: losimo@gmail.com.

## Measures

Measures will be proportional and progressive: warning, temporary block or permanent ban. Serious cases may receive an immediate response.

## Final

If you have a technical disagreement, explain it clearly and with data. If you have a conduct complaint, communicate it through a private channel with as little exposure as possible.
